# Phase 3: Review

You are starting the **Review** phase. The Plan phase must be completed first (all `analysis/*.md` files should exist).

## Review Checklist

### 1. Coherence Matrix

Generate a coherence matrix that maps brand elements (rows) against communication touchpoints (columns). Each cell gets a score (high/medium/low) with a brief annotation.

**Rows:** Mission, each brand value, archetype, Big Idea, client promise
**Columns:** Visual style, tone of voice, digital platform, outdoor advertising, product line, client service

Create `analysis/coherence_review.md` with:
- The scored matrix
- Gap analysis (any cells below 67%?)
- Recommendations for closing gaps

### 2. Cross-Block Alignment

Verify that:

| Check | What to verify |
|-------|---------------|
| **Values ↔ Messages** | Each message maps to a specific value |
| **Archetype ↔ Tone** | Brand voice examples match archetype personality |
| **Big Idea ↔ Slogans** | Each slogan derives from the Big Idea formula |
| **Values ↔ Architecture** | Each sub-brand emphasizes a value + the core value |
| **Colors ↔ Messages** | Secondary colors map to message system |
| **Graphic device ↔ Metaphor** | Visual elements reinforce the brand concept |

### 3. Brief Compliance

Check deliverables against the original assignment:
- Does the analytical block cover SWOT and competitive analysis?
- Does the strategic block have mission, values, archetype, Big Idea, architecture?
- Does the communication block have key idea, 3 messages, 3-5 slogans, sample layout?
- Does the visual block have identity concept, color, typography, graphic device, applied mockups?
- Is everything within page/slide limits?

### 4. Competitive Distinctiveness

For each competitor, verify the brand doesn't accidentally overlap:
- Different archetype quadrant?
- Different tone of voice?
- Different visual territory?
- Distinct value proposition?

## Output

Create `analysis/review_report.md` with:
- Overall coherence score (%)
- Per-check pass/fail with notes
- Priority fixes (if any)
- Sign-off recommendation

## Important

- Be critical but constructive
- Flag actual gaps, don't invent problems
- If everything passes, recommend proceeding to Execute
