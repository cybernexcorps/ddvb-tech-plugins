# Visual Generation for Brand Strategy

## Architecture

All figures are generated by Python scripts using matplotlib. A shared font module provides typography consistency.

### Font Module (`scripts/fonts.py`)

```python
from fonts import font_props

# Usage: font_props(role, size) → matplotlib FontProperties
# Roles: title, subtitle, header, body, body_medium, body_bold, label, annotation
ax.text(x, y, "Title", fontproperties=font_props("title", 24), color="#0D0D0D")
```

The font module scans the user's font directory for the project typeface and registers all weights with matplotlib.

### Two Rendering Modes

**Transparent mode** — for figures on white/light slide backgrounds:
```python
fig.patch.set_alpha(0)
ax.set_facecolor("none")
fig.savefig(path, transparent=True, dpi=250, bbox_inches="tight")
```
- Text colors: dark (#0D0D0D, #44546A)
- Borders: light (#D9D9D9)
- Use for: SWOT, competitive map, brand wheel, color system, typography

**Dark mode** — for creative mockups that ARE dark by design:
```python
fig.patch.set_facecolor("#0D0D0D")
fig.savefig(path, facecolor="#0D0D0D", transparent=False, dpi=250, bbox_inches="tight")
```
- Text colors: white, gold (#FDB71C)
- Use for: ad mockups, graphic device demos, applied identity

### Color Palette Pattern

Define all colors as module-level constants matching the brand:
```python
GOLD = "#FDB71C"
BLACK = "#0D0D0D"
BLUE = "#1565C0"    # Message 1
VIOLET = "#6A1B9A"  # Message 2
AMBER = "#E65100"   # Message 3
GREEN = "#2E7D32"   # Strengths
RED = "#C62828"     # Weaknesses
```

### Common Pitfalls

- Never use matplotlib default fonts — always `fontproperties=font_props(...)`
- Never use `fontfamily="monospace"` — it bypasses the brand font
- Gold text on white background is unreadable — use gold only on dark or as fills
- `FancyBboxPatch` with `boxstyle="round,pad=0.08"` for all card shapes
- Always set `zorder` explicitly to control layering
